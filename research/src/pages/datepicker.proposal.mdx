# Datepicker Component Specification

## Overview <a href="#overview" id="overview"></a>

The `<datepicker>` component is a control composed of an input and modal calendar, which enabled users to pick a particular date or range of dates.

### Background <a href="#background" id="background"></a>

_Relevant historical or background information, related existing issues, etc._

### Use Cases <a href="#use-cases" id="use-cases"></a>

The `<datepicker>` control is primarily used to select a date or range of dates in forms and/or filters. For example when ordering a delivery user may be provided with a `<datepicker>` to select desirable date of delivery. Or for example when looking for events occuring in a city, user may be provided with a `<datepicker>` to select the range of dates to filter the events shown.

### Accessibility <a href="#accessibility" id="accessibility"></a>

Accessibility recommendations for this component are described in [WAI-ARIA example](https://www.w3.org/TR/wai-aria-practices/examples/dialog-modal/datepicker-dialog.html):

> ... a date input field and a button that opens a date picker that implements the dialog design pattern. The dialog contains a calendar that uses the grid pattern to present buttons that enable the user to choose a day from the calendar. Choosing a date from the calendar closes the dialog and populates the date input field. When the dialog is opened, if the input field is empty, or does not contain a valid date, then the current date is focused in the calendar. Otherwise, the focus is placed on the day in the calendar that matches the value of the date input field.
